# C++ задачки

## Easy

### aboba.cpp
**Вывод:** `A A B ~B ~A ~A`
Конструкторы вызываются сверху вниз (сначала Base, потом Derived), деструкторы — в обратном порядке. `b` уничтожается раньше `a`.

### bracket_is_bracket.cpp
**Вывод:** `3 2`
`vector(3, 5)` — 3 элемента со значением 5. `vector{3, 5}` — список инициализации из двух элементов. Скобки решают всё.

### brave_ternary.cpp
**Вывод:** `10 2`
Тернарный оператор возвращает lvalue, если оба варианта — переменные одного типа. `(true ? a : b) = 10` присваивает 10 переменной `a`.

### did_you_forget_smth.cpp
**Вывод:** `5`
`auto` срезает ссылку — `y` становится копией, а не алиасом. Изменение `y = 10` не затрагивает `x`.

### float_hell.cpp
**Вывод:** `0`
Вещественные числа хранятся приближённо. `0.1 + 0.2` = `0.30000000000000004` ≠ `0.3`.

### long_arrow.cpp
**Вывод:** `4 3 2 1 0`
`-->` — не оператор. Это `(x--) > 0`: постдекремент и сравнение. Цикл идёт пока x ещё не нуль, но печатает уже уменьшенное значение.

### minus_and_minus_is_plus.cpp
**Вывод:** `0`
`x -= x -= 3`: сначала `x -= 3` → x=7, потом `x -= 7` → x=0.

### o_o.cpp
**Вывод:** `0 0`
Цепочка сравнений не работает как в математике. `3 > 2 > 1` = `(3>2) > 1` = `1 > 1` = `0`. Аналогично со второй частью.

### too_much_letters.cpp
**Вывод:** `12 12 12 12`
`f` принимает по значению — ничего не меняет. `g` принимает по ссылке — меняет оригинал. `x` и `y` — ссылки на `a` и `b`, поэтому `g(x)` эквивалентно `g(a)`. Каждая переменная инкрементируется дважды: 10 → 11 → 12.

## Medium

### forget_git_pull.cpp
**Вывод:** `1`
`[=]` захватывает переменные по значению в момент создания лямбды. Лямбда сохранила `x = 1`. Последующее `x = 2` её не касается.

### global_granny.cpp
**Вывод:** `100`
Виртуальное наследование даёт одного общего `Granny`. Порядок вызова конструкторов: сначала `Dad` (a=1000), потом `Mom` (a=100). Побеждает последний.

### infinity_is_not_a_limit.cpp
**Вывод:** `greater`
`v.size()` возвращает `size_t` — беззнаковый тип. При сравнении `int(-1)` конвертируется в `size_t` и становится очень большим числом.

### laaazy_compiler.cpp
**Вывод:** `3`
Короткое замыкание `||`: если левая часть истинна, правая не вычисляется. `a-- > 3` истинно (a=4), поэтому `++a` не выполняется. a = 3.

### ravnyays_smirno.cpp
**Вывод:** `12 8`
Процессор требует выравнивания `int` по 4 байтам. В `struct A` после первого `char` вставляется 3 байта паддинга, и ещё 3 в конце. В `struct B` два `char` идут подряд — паддинга меньше.

## Hard

### broken_move.cpp
**Вывод:** `hello hello`
`std::move` на `const` возвращает `const string&&`. Move-конструктор не принимает `const&&`, срабатывает копирование. Строка не перемещается, `s` остаётся нетронутым.

### computer_vision.cpp
**Вывод:** `1`
`foo() const` в Base и `foo()` без const в Derived — это разные функции. Derived не переопределяет Base, а скрывает. Через `Base&` вызывается Base::foo.

### darova.cpp
**Вывод:** `ello`
Строковый литерал распадается в `const char*`. `1 + "hello"` — арифметика указателей, сдвиг на 1. Результат указывает на подстроку "ello".

### division_by_zero_is_ok.cpp
**Вывод:** `0`
`0.0 / 0.0` = NaN. По стандарту IEEE 754 NaN не равно ничему, даже самому себе. `x == x` → false.

### hide_and_seek.cpp
**Вывод:** `3`
`f(T, T)` более специализированна, чем `f(T, U)`, поэтому выбирается она. Явная специализация `f<int, int>` — это специализация `f<T, U>` (объявленной первой), а не `f<T, T>`. Итого: вызывается `f<T, T>` без специализации → 3.

### keyboard_rush.cpp
**Вывод:** `42`
`*&*p**&*q` = `(*p) * (*q)` = `6 * 7 = 42`. Выглядит пугающе, работает просто: `*&` взаимно уничтожаются, `**` — умножение.

### operations_race.cpp
**Вывод:** `0`
Приоритет `==` выше `&`. Поэтому `x & 1 == 0` читается как `x & (1 == 0)` = `2 & 0` = `0`. Кажется, что проверяем чётность — но нет.

### sizeof_distraction.cpp
**Вывод:** `4 0`
`sizeof` не вычисляет своё выражение — это compile-time операция. `x++` никогда не выполняется, `x` остаётся 0.

### tuda_suda.cpp
**Вывод:** `1`
Виртуальная диспетчеризация выбирает `Derived::f()` — верно. Но дефолтные аргументы разрешаются статически по типу указателя `Base*`. Функция Derived, аргумент из Base.

### we_are_losing_him.cpp
**Вывод:** `8`
Массив в параметре функции молча превращается в указатель. `sizeof(arr)` внутри функции = размер указателя = 8 байт.

## Insane

### cyber_swap.cpp
**Задача на реализацию:** поменять два числа местами без временной переменной.
Классическое решение — XOR-swap: `a ^= b; b ^= a; a ^= b;`.

### god_loves_three.cpp
**Вывод:** `0 1`
`i&&&i` — это `i && (&i)`. `&i` — указатель на `i`, всегда ненулевой (true). Результат зависит только от `i`.

### imposter.cpp
**Вывод:** `C 67`
`int8_t` — это typedef от `char`. `cout << char(67)` печатает символ 'C'. `int16_t` — short, печатается как число 67.

### keyboard_error.cpp
**Вывод:** `42`
`p->*&S::x` — разыменование указателя на член, `1[p]` = `p[1]`. Итого: `a[0].x * a[1].x` = `6 * 7 = 42`. Два экзотических оператора — один результат.

### smth_is_better_than_nothing.cpp
**Вывод:** `1 10`
По стандарту C++ объектов с нулевым размером не бывает — пустой struct занимает 1 байт. Массив из 10 штук = 10 байт.

### under_construction.cpp
**Вывод:** `AB`
Во время конструктора vtable указывает на текущий класс, а не на самый производный. `A()` вызывает свою `f`, `B()` — свою. `C::f` не доступна ни разу.

## Demon

### double_lookup.cpp
**Вывод:** `global`
Двухфазный поиск имён в шаблонах: не зависящее от параметра имя `f` ищется в момент парсинга шаблона — находится глобальная `f`. Чтобы найти `Base<T>::f`, нужен `this->f()`.

### double_trouble.cpp
**Вывод:** `0 1 2`
Копия лямбды — другой объект, но того же типа. `static` переменная принадлежит типу, а не экземпляру. Все копии лямбды делят один счётчик `n`.

### positive_zero.cpp
**Вывод:** `1`, затем `0`, затем невидимый символ
`a + 1` в сравнении промоутится до `int(256)` → больше нуля. Но `b = a + 1` обрезается до `unsigned char(0)`, поэтому `b > 0` уже false. `cout << b` печатает `'\0'` — нулевой символ, невидимый.
